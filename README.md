# Telco Customer Churn Prediction Practice

## 概要

本プロジェクトは、Kaggle形式の顧客解約予測タスクを題材に、機械学習による二値分類モデルの構築を練習したものです。
目的変数は顧客が解約するかどうかを表す `Churn` であり、契約情報・利用サービス・料金情報などをもとに解約確率を予測します。

本取り組みでは、生特徴量設計、複数モデルの学習、OOF予測、スタッキングなど、機械学習コンペで用いられる一連の流れを学習・実装しました。
あくまで学習・練習目的のプロジェクトであり、実務利用を目的としたものではありません。

## 使用データ

主に以下のデータを想定しています。

* `train.csv`
* `test.csv`
* `sample_submission.csv`
* `WA_Fn-UseC_-Telco-Customer-Churn.csv`

Kaggle環境での実行を想定し、ローカルパスとKaggle上の入力パスの両方に対応できるようにしています。

## 主な処理内容

### 1. データ読み込み・前処理

`Churn` を `Yes / No` から `1 / 0` に変換し、二値分類タスクとして扱います。
また、`TotalCharges` は数値型に変換し、欠損値については中央値で補完しています。

### 2. 特徴量生成

顧客の契約状況や料金構造を表すために、以下のような特徴量を作成しています。

* 月額料金と総支払額の関係
* 契約期間と料金の組み合わせ
* 利用しているサービス数
* セキュリティ系サービスの利用数
* ストリーミング系サービスの利用数
* 新規顧客かどうか
* 長期契約かどうか
* 月単位契約かどうか

また、数値の桁・丸め・剰余に関する特徴量や、カテゴリ変数同士を組み合わせた bi-gram / tri-gram 特徴量も作成しています。

### 3. original dataset を用いた統計特徴量

追加データである original dataset を利用し、カテゴリごとの解約率や件数を特徴量として追加しています。
これにより、各カテゴリが過去データ上でどの程度解約と関係しているかをモデルに反映させています。

### 4. 分布に基づく特徴量

解約者・非解約者それぞれの `TotalCharges` 分布と比較し、各顧客がどちらの分布に近いかを表す特徴量を作成しています。
これにより、単純な金額情報だけでなく、解約傾向との相対的な位置関係をモデルに学習させることを狙っています。

### 5. 顧客セグメント特徴量

KMeans を用いて、料金・契約・サービス利用状況が似ている顧客をクラスタリングしています。
作成したクラスタ番号とクラスタ中心からの距離を特徴量として追加し、顧客セグメントごとの解約傾向を捉えることを目的としています。

### 6. Target Encoding

カテゴリ変数に対して、fold 内でさらに inner CV を用いた Target Encoding を行っています。
これは、学習データ側で目的変数の情報が直接漏れることを防ぐためです。

### 7. モデル学習

以下のモデルを使用しています。

* XGBoost
* CatBoost
* LightGBM
* Logistic Regression

Stratified K-Fold による交差検証を行い、各foldでOOF予測を作成しています。
評価指標には AUC を使用しています。

### 8. スタッキング

各ベースモデルの予測値をもとに、メタモデルとして Logistic Regression を学習しています。
また、各モデルの予測平均・標準偏差・最大値・最小値・予測幅などもメタ特徴量として使用し、モデル間の予測のばらつきも活用しています。

* ドメイン知識を反映した特徴量生成
* カテゴリ変数に対する Target Encoding
* OOF予測を用いたモデル評価
* 複数モデルを組み合わせるアンサンブル手法
* スタッキングによる予測精度向上の考え方
* 生成AIを学習補助として活用する方法
